# NDP Proxy — שרת ביניים לנדרים פלוס (Netlify)

שרת קטן (Netlify Function) שמטרתו לעקוף את חסימת ה-CORS של נדרים פלוס,
כדי שאתר ניהול בית הכנסת יוכל להתחבר אליהם.

## פריסה (Deploy) — שלב אחר שלב

### 1. יצירת ריפו ב-GitHub
אם כבר יש לך ריפו מהניסיון הקודם (Vercel) — אפשר להשתמש בו, רק תמחק את
התיקייה `api` ותעלה במקומה את התיקיות והקבצים מהזיפ הזה (`netlify`, `netlify.toml`, `package.json`).

אם אתה מתחיל מאפס:
1. כנס ל-https://github.com/new וצור ריפו חדש
2. לחץ "uploading an existing file" / "העלאת קובץ קיים"
3. גרור לשם את כל התוכן מהזיפ הזה (כולל תיקיית `netlify/functions`)
4. Commit changes

### 2. חיבור ל-Netlify
1. כנס ל-https://app.netlify.com
2. הירשם/התחבר עם **GitHub** (לא צריך אימות טלפון בדרך כלל)
3. לחץ "Add new site" → "Import an existing project"
4. בחר "Deploy with GitHub" ואשר את ההרשאות
5. בחר את הריפו שלך מהרשימה
6. השאר את ההגדרות כברירת מחדל ולחץ "Deploy"
7. תוך דקה תקבל כתובת כמו: `https://random-name-12345.netlify.app`

### 3. כתובת ה-API שלך
```
https://random-name-12345.netlify.app/.netlify/functions/nedarim
```

⚠️ שים לב: אצל Netlify הנתיב הוא `/.netlify/functions/nedarim` ולא `/api/nedarim` כמו ב-Vercel.

שלח לי את הכתובת המלאה שקיבלת ואני אעדכן את הקוד באתר שלך.

## בדיקה מהירה
אחרי הפריסה, אפשר לבדוק שהשרת חי על ידי כניסה לכתובת:
```
https://random-name-12345.netlify.app/.netlify/functions/nedarim
```
(בדפדפן רגיל — אמורה לחזור שגיאת "רק בקשות POST מותרות", זה תקין וסימן שהפונקציה פעילה)

## אבטחה
- מספר הקופה והסיסמה עוברים דרך השרת אך לא נשמרים בו.
- מומלץ בעתיד להגביל את `Access-Control-Allow-Origin` בקובץ `netlify/functions/nedarim.js` לדומיין של האתר שלך בלבד.

## מיילים מורשים להתחברות
כדי שלא כל אחד יוכל לבקש קישור התחברות חד-פעמי למייל, יש טבלת `allowed_emails` ב-Supabase —
רק מיילים שנמצאים בה יקבלו קישור (ראה `supabase-schema.sql`).

**הפעלה ראשונית:** אחרי הרצת `supabase-schema.sql`, הרשימה ריקה — אף אחד לא יוכל להתחבר במייל.
הוסף את המייל הראשון שלך (המנהל) ידנית דרך Supabase → SQL Editor:
```sql
insert into allowed_emails (email, label) values ('your-email@example.com', 'מנהל ראשי');
```
לאחר שתתחבר עם המייל הזה באתר, תוכל לנהל את שאר הרשימה (להוסיף/להסיר מיילים) ישירות
מהמסך "🔐 מיילים מורשים להתחברות" בלשונית "נדרים פלוס" באתר עצמו — בלי לחזור ל-Supabase.

## תזכורות תרומה (מעקב אחרי תורמים חד-פעמיים)

ניתן לרשום תרומה חד-פעמית של תורם, ולתזמן תזכורת אוטומטית שתישלח במייל (עד 2 כתובות) בתאריך
עתידי שתבחר — לדוגמה "בעוד 6 חודשים תזכיר לי לפנות שוב לתורם הזה". התזכורות מנוהלות מתוך
לשונית "👥 קהילה" באתר, בכרטיס "💰 תרומות חד-פעמיות ותזכורות".

**הפעלה — שלב חובה כדי שהמייל באמת יישלח:**
1. הירשם בחינם ל-[Resend](https://resend.com) (שירות שליחת מיילים, יש להם תוכנית חינמית נדיבה).
2. ב-Resend, צור API Key (Dashboard → API Keys → Create API Key).
3. ב-Netlify, היכנס לאתר שלך → Site configuration → Environment variables, והוסף:
   - `RESEND_API_KEY` — המפתח שיצרת ב-Resend.
   - `REMINDER_FROM_EMAIL` — כתובת השולח. אם לא הגדרת דומיין משלך ב-Resend, אפשר להשאיר את ברירת
     המחדל `onboarding@resend.dev`, אך שימו לב: בלי אימות דומיין, Resend מאפשר לשלוח רק לכתובת
     המייל שאיתה נרשמת ל-Resend. כדי לשלוח לכל כתובת (כולל לשני המיילים שתגדיר באתר), צריך
     לאמת דומיין משלך ב-Resend (Domains → Add Domain) ולהשתמש בכתובת מהדומיין הזה, למשל
     `tizkurot@beit-knesset-shelanu.org`.
4. ודא ש-`SUPABASE_SERVICE_ROLE_KEY` כבר מוגדר ב-Environment Variables של Netlify (אותו מפתח
   שכבר נדרש לפונקציית השידור החוזר האוטומטי — אם הוא כבר עובד, אין צורך להגדיר שוב).
5. הפונקציה `send-donor-reminders` רצה אוטומטית כל יום (ראה `netlify.toml`) ובודקת אם הגיע
   הזמן לשלוח תזכורת. אפשר גם להריץ אותה ידנית לבדיקה דרך הכתובת:
   `https://your-site.netlify.app/.netlify/functions/send-donor-reminders`

שימו לב: ניהול התזכורות (הוספה/מחיקה) דרך האתר דורש התחברות במייל (לא בסיסמה הראשית) —
אותה דרישה הקיימת כבר עבור הגדרות השידור החוזר האוטומטי.

## קו טלפוני ימות המשיח (מכירות + הוצאות ע"י גבאים)

מאפשר לציבור להזמין/לקנות בטלפון, ולגבאים להזין הוצאות דרך תפריט מוגן קוד PIN — הכל נכנס
אוטומטית לדוח הכספי באתר. ניהול מלא (קטלוג מכירות, רשימת גבאים, PIN, הודעה קולית) מתבצע
מתוך לשונית "☎️ ימות המשיח" באתר עצמו.

**הפעלה:**
1. ב-Netlify → Environment variables, הוסף:
   - `IVR_API_SECRET` — מחרוזת סודית שרק אתה יודע (למשל 32 תווים אקראיים) — מגינה על ה-API מפני שימוש לא מורשה. זה **לא** הטוקן של ימות המשיח עצמו — זה סוד פנימי בינך לבין השרת שלך.
   - ודא ש-`SUPABASE_SERVICE_ROLE_KEY` כבר מוגדר (נדרש כבר לפיצ'ר התזכורות).
2. הרץ שוב את `supabase-schema.sql` המעודכן (מוסיף טבלת `ivr_kv`, בטוח להריץ שוב).
3. באתר, בלשונית "☎️ ימות המשיח", הזן את אותו `IVR_API_SECRET` ולחץ "שמור מפתח וטען הגדרות".
   הגדר שם את קטלוג המכירות, רשימת מספרי הגבאים, קוד ה-PIN, **ואת מפתח ה-API של ימות המשיח עצמו**
   (שדה "מפתח API של ימות המשיח") — זה הטוקן שלך אצל ימות (בפורמט `מספר_מערכת:סיסמה`, נמצא
   בממשק הניהול שלך אצלם תחת "מפתח למפתחים"/API). זה נדרש רק כדי להשמיע רשימות ארוכות בקול
   (כמו "5 הוצאות אחרונות") — בלעדיו הפיצ'ר הזה פשוט לא יעבוד, שאר המערכת כן.
   לחלופין, אפשר גם להגדיר אותו כ-`YEMOT_TOKEN` ב-Environment Variables של Netlify — אבל
   הרבה יותר נוח לעדכן אותו ישירות מהאתר, בלי לגעת ב-Netlify בכלל.
4. **בחשבון ימות המשיח שלך** — יש לבנות שלוחות מסוג `type=api` שמצביעות לכתובת הפונקציה
   (`.../.netlify/functions/ivr`), לפי מבנה השלוחות המדויק המתועד כהערה בראש הקובץ
   `netlify/functions/ivr.js` (שם השלב הנדרש בכל שלוחה, איזו שלוחה מפנה לאיזו, וכו').
   זהו שלב שדורש גישה בפועל לחשבון ימות המשיח שלך ולא ניתן לבצע מרחוק — מומלץ להיעזר
   בתמיכה הטכנית של ימות המשיח או בפורום המפתחים שלהם (freeivr.co.il) אם נתקעים בהגדרה.
5. לאחר שהשלוחות מוגדרות, בדוק שיחת ניסיון. ההזמנות/הוצאות ייכנסו ללשונית "☎️ ימות המשיח"
   באתר, ומשם אפשר ללחוץ "🔄 סנכרן לכספים" (או לחכות לרענון האוטומטי כל 5 דקות אם אתה
   מחובר גם לנדרים פלוס) כדי שיופיעו גם בדוח הכספי הראשי.

**קטלוג מכירות מאקסל:** בכרטיס הקטלוג יש כפתור "⬆ ייבוא מאקסל" — קובץ עם עמודות "שם מוצר",
"מחיר", ואופציונלית "יחידה" (כתבו "קילו" או "ק"ג" למוצרים הנמכרים לפי משקל).

**הנחת הו"ק לתורמים קבועים:** למוצר שמוגדר "לקילו", המערכת בודקת בזמן השיחה אם מספר
הטלפון של המתקשר משויך להוראת קבע פעילה בנדרים פלוס בסכום שהגדרת (ברירת מחדל ₪100+)
וללא סירוב פתוח כרגע — אם כן, מפחיתה את סכום ההנחה שהגדרת (ברירת מחדל ₪2) מכל קילו.
רשימת הזכאים מתעדכנת אוטומטית בכל סנכרון עם נדרים פלוס, ודורשת שתהיה מחובר גם ללשונית
"💳 נדרים פלוס" כדי שתהיה עדכנית. שימו לב שזו בדיקה מבוססת על המצב הנוכחי של ההוראת
הקבע (לא סריקה מדויקת חיוב-אחר-חיוב של 3 החודשים האחרונים).
